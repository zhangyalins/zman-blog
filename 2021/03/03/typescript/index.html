什么是TypeScript？

TypeScript是添加了类型系统的Javascript，适用于任何规模的项目。
Typescript是一门静态类型、弱类型语言
Typescript是完全兼容Javascript的，它不会修改Javascript运行时的特性
Typescript可以编译为Javascript，然后运行在浏览器，Node.js等任何能运行的Javascript环境中。

Typescript和Javascript的区别：

1.Typescript用于解决大型项目的代码复杂性
可以在编译期间发现并纠正错误
强类型，支持静态和动态类型
支持模板、泛型和接口
需要编译成Javascript代码
2.Javascript是一种脚本语言，用于创建动态网页
作为一种解释型语言，只能在运行时发现错误
弱类型，没有静态类型选项
可以直接在浏览器中使用
不支持模块、泛型和接口

安装Typescript

npm install -g typescript

编译Typescript

tsc helloworld.ts // 编译后ts转js helloworld.ts => helloworld.js

使用ts-node编译TypeScript

npm install -g ts-node
命令
ts-node demo1.ts

Typescript分为静态类型和对象类型

Typescript静态类型

在Typescript中，使用:指定变量的类型。 如果使用了静态类型，那么变量的类型将不能改变(Typescript在编译时对类型进行静态检查，发现错误，在编译时就会报错,但还是会生成编译结果)

Boolean类型
let isDone:boolean = false
使用构造函数Boolean创造的对象不是布尔值

let createBoolean : boolean = new Boolean(1); //报错(事实上new Boolean()返回的是Boolean对象)

直接调用Boolean也可以返回一个boolean类型

let createBoolean : boolean = Boolean(1)

Number类型

String类型

Array类型
let list:number[] = [1,2,3]

空值

javascript没有空值的概念，在Typescript中，可以用void表示没有任何返回值,声明void类型变量只能将它赋值为undefined和null

null和undefined

与void的区别是，undefined和null是所有类型的子类型。undefined类型的变量，也可以赋值给number类型的变量

任意值类型

任意值用来表示允许赋值为任意类型,当一个变量为任意值时，对它的任何操作返回的内容的类型都是任意值

未声明类型的变量

变量在声明时未指定其类型，那么它会被识别为任意值类型

类型推论

在没有为变量指定类型时，TS会自动推断该变量的类型；如果在定义时没有赋值，则会推断成any类型。

联合类型

表示取值可以为多种类型中的一种，使用|分割每一个类型

当Typescript不确定一个联合类型的变量到底是什么类型的时候，只能访问此联合类型的所以类型里的共有属性和方法。

联合类型在变量被赋值时，会根据类型推论规则推断出一个类型

对象类型

对象类型——接口

在Typescript中，使用接口(interfaces)来定义对象的类型
接口是对行为的抽象，而具体行动由类(classes)去实现

定义的变量比接口少一些/多一些属性是不允许的

赋值时，变量的属性必须和接口的形状保持一致

可选属性

有时不希望完全匹配，可以使用可选属性

任意属性

有时一个接口需要允许有任意的属性

方法

一旦定义了任意属性，确定属性和可选属性的类型必须是它类型的子集

只读属性

有时需要对一些字段设置只能在创建的时候赋值，那么可用readonly定义只读属性

类型别名和接口的区别：类型别名可以直接给类型，但接口必须代表对象

数组类型

类数组

常见的数组表示法，使用any表示数组中允许出现任意类型

元组

将数组中的每个元素类型的位置固定住，叫做元组
const arr6 : [string,number,string] = ['1',25,'3'];
